# Final Coarse Control Program Audit

Status: `analysis_only`
Classification: `coarse_control_program_candidate`
Translation delta: `NONE`
Plaintext claim: `False`
Case reopened: `False`

## Question

After exact `type:length` failed and length was factored into coarse bucket plus residual innovation, this audit tests whether the coarse `type:length_bucket` stream can be encoded or generated by a target-free sequential program. Per-book operation count is granted.

This is not a book generator: exact within-bucket length residuals, literal payload, copy hints, seed books, and row0 remain external.

## Result

- Vocabulary: `12` coarse symbols.
- Best model: `op_count` with features `op_count_bucket`.
- Best model bits over prefix/suffix cutoffs: `1512.974`.
- Saving vs uniform coarse control: `254.413` bits.
- Same-multiset shuffled p95 saving: `116.843` bits.
- Top1 hits: `88/493`.
- Promoted models: `['prev_symbol', 'op_pos', 'remaining_ops', 'op_count', 'prev_x_pos', 'prev_x_remaining_ops', 'count_x_pos']`.
- Generator-promoted coarse models: `['prev_symbol', 'op_pos', 'remaining_ops', 'op_count', 'prev_x_pos', 'prev_x_remaining_ops', 'count_x_pos']`.

The result survives same-multiset shuffled controls. This matters because the global marginal distribution alone is not promoted; the promoted models use operation count, operation position, remaining operations, or previous coarse symbol.

## Decision

- `coarse_control_program_candidate`.
- `type:length_bucket` is no longer only an undifferentiated atlas field; it has a prefix-generalizing coarse sequential controller candidate.
- The candidate still grants per-book op count and does not produce exact length residuals.
- `row0`, translation, plaintext, and the compression bound remain unchanged.

## Remaining External Fields

- literal innovation tape payload and schedule
